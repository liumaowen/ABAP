# Object Scope

## 环境说明
当前处于开发环境（DEV）。
原则上允许修改开发环境中的对象，但所有可运输对象的变更必须绑定到 Transport Request。

## 允许 AI 操作的对象
- DEV 环境中的开发对象
- Z* / Y* 自定义对象
- 当前任务明确涉及的类、程序、函数组、接口、CDS、BDEF、SRVD、表、消息类等

## 约束
- 任何可运输对象的修改前，必须确保已创建或绑定 Transport Request
- 修改前优先确认对象所属包、传输层和影响范围
- 如果对象属于标准对象或跨系统共享对象，修改前必须先确认
- 如果涉及激活、发布、删除、数据模型变更或权限变更，必须先确认

## 修改原则
- 优先最小变更
- 优先保持现有结构和命名
- 每次只改一个明确范围内的对象
- 修改后必须给出验证步骤和回滚思路

## 请求要求
- 每次修改都必须挂接到 Transport Request
- 如果当前没有请求，先创建请求，再执行修改
- 修改完成后，说明请求号、修改内容和影响范围
