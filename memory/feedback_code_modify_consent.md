## 代码修改确认流程

任何代码修改都需要用户明确同意：
1. 解释你计划修改什么以及为什么
2. 等待用户确认后再执行
3. 不要在没有明确批准的情况下修改代码

### 例外情况
- 用户已经明确说"继续"或"执行"
- 简单的格式调整（用户已同意整体方案）

**Why:** 确保用户对代码变更有完全控制权
**How to apply:** 每次修改代码前，先向用户说明计划并等待确认
